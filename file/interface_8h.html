File interface.h¶

Available functions in the ZeroMQ XOP

This file is part of the ZeroMQ-XOP project and licensed under BSD-3-Clause.

  • All functions returning a variable always return zero

  • Errors are signalled via RTE, use 

    zeromq_bind("blah"); err = GetRTError(1)
    for programmatically handling the error. The error values are defined at ZeroMQErrorCodes.

  • The XOP applies the following settings:

    • Message pattern: DEALER/ROUTER

    • DEALER/ROUTER: ZMQ_SNDTIMEO = ZMQ_RCVTIMEO = 0

    • DEALER/ROUTER: ZMQ_LINGER = 0

    • ROUTER: ZMQ_ROUTER_MANDATORY = 1

    • ROUTER: ZMQ_MAXMSGSIZE = 1024 byte

    • DEALER: ZMQ_IDENTITY non-empty string

  • For compatibility reasons with the REQ socket, DEALER sockets have to always send two frames: An empty one and one with the payload.

Message handler

The XOP implements a threaded message handler. This message handler can be started and stopped with the following commands.

variable zeromq_handler_start()¶
variable zeromq_handler_stop()¶

Will be implicitly called on Igor close.

Functions

variable zeromq_stop()¶

Stop all listening binds and disconnect outgoing connections, stops also the message handler. Internally closes also the sockets.

Will be implicitly called on Igor close.

variable zeromq_set(variable flags)¶

Allow to adjust various runtime features.

The flags can be or’ed.

Parameters

  • flags: One of ZeroMQSetFlags

variable zeromq_server_bind(string localPoint)¶

Start listening on the given TCP port.

Parameters

  • localPoint: transport protocol and address, something like tcp://127.0.0.1:5670 should be used, the TCP/IP port must be > 1024 and unused, 5670 is a good default choice [standard port for FileMQ]

variable zeromq_server_send(string identity, string msg)¶

Send a messsage as the ROUTER socket

Parameters

  • identity: client identifier

  • msg: Message to send

string zeromq_server_recv(string *identity)¶

Receive a message

Implemented using a blocking wait albeit abortable from within Igor Pro.

Return

received message

Parameters

  • [out] identity: client identifier, required for sending a message back via zeromq_server_send()

variable zeromq_client_send(string msg)¶

Send a messsage as the DEALER socket

Parameters

  • msg: Message to send

variable zeromq_client_connect(string remotePoint)¶

Connect to a ZMQ server, e.g. a different Igor instance running as a server.

Parameters

  • remotePoint: Protocol and address of the server, usually something like tcp://127.0.0.1:5670

string zeromq_client_recv()¶

Receive a message as the DEALER socket

Implemented using a blocking wait albeit abortable from within Igor Pro.

Return

received message
